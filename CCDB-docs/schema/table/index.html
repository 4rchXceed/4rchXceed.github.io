Table schema

The json schema for a table is this:

{
  "schema": {
    "current_sysid": 1,
    "name": "item",
    "indexes": [
      {
        "field": "id",
        "type": "pk"
      },
      {
        "field": "item_id",
        "type": "unique"
      },
      {
        "field": "id",
        "type": "ai",
        "start": 1,
        "current": 1
      }
    ],
    "fields": {
      "id": {
        "type": ["integer"],
        "nullable": false,
        "default": null
      },
      "item_id": {
        "type": ["varchar", 50],
        "nullable": false,
        "default": null
      },
      "item_count": {
        "type": ["integer"],
        "nullable": true,
        "default": 0
      }
    }
  },
  "data": [
    {
      "?sysid?": 1,
      "id": 1,
      "item_id": "minecraft:stone",
      "item_count": 64
    },
    {
      "?sysid?": 2,
      "id": 2,
      "item_id": "minecraft:dirt",
      "item_count": 32
    }
  ]
}

Explanation

  • schema: The structure of the table
  • current_sysid: The current system id, used for internal purposes (auto-incrementing)
  • name: The name of the table (unused, left because idk why)
  • indexes: The indexes of the table, an array of objects
  • fields: The fields of the table, an object
  • data: The data of the table, an array of objects (each object is a row)

Indexes

Primary key (pk)

The primary key index is used to uniquely identify each row in the table. It is defined by setting the type to pk and specifying the field that will be used as the primary key.

Example:

{
  "field": "id",
  "type": "pk"
}

Unique key (unique)

The unique key index is used to ensure that all values in a column are different from each other. It is defined by setting the type to unique and specifying the field that will be used as the unique key.

Example:

{
  "field": "item_id",
  "type": "unique"
}

Auto-incrementing key (ai)

The AI key index is used to automatically generate a unique value for each new row inserted into the table. It is defined by setting the type to ai. The field specifies the column that will be auto-incremented, start specifies the starting value, and current keeps track of the current value.

Example:

{
  "field": "id",
  "type": "ai",
  "start": 1,
  "current": 1
}

Fields

Each field in the fields object represents a column in the table.

  • key: The name of the column
  • type: The data type of the column (e.g., integer, varchar) in an array. The first element is the type name, and the second element (optional) can specify additional parameters (e.g., maximum length for varchar).

Supported

  • int: A whole number (e.g., 1, 2, 3)
  • varchar: A string of characters (e.g., "hello", "world"). The maximum length can be specified as a second element in the type array (e.g., ["varchar", 50] for a maximum length of 50 characters).
  • float: A number with a decimal point (e.g., 3.14, 2.718)
  • boolean: A true or false value (e.g., true, false)
  • null: Do you really want to use it?
  • text: A varchar, but with no maximum length (e.g., "This is a long text that can be of any length.")

Nullable

The nullable property allows the value of the field to be null.

Default

The default property specifies the default value for the field if no value is provided during insertion.

Data

  • ?sysid?: This is an internal field used for auto-incrementing purposes. It should not be included in the data when inserting new rows, as it will be automatically generated by the system.
  • Each object in the data array represents a row in the table, with key-value pairs corresponding to the column names and their respective values.
